# 什么是微服务
很小很小的服务，一个服务对应一个单一的功能，单独部署

# 微服务与分布式的区别
微服务是架构设计，分布式是部署方式，概念不同

服务之间的逻辑不要偶尔，比如商品库服务的逻辑不要写在物流服务里面

微服务设计原则
- 单一职责原则，每个服务只要实现自己的业务
- 服务自治

微服务设计原则，解决方案

采用微服务架构来构建系统，会带来清晰的业务划分和可扩展性。

服务化的核心就是将传统的一站式应用拆分成多个服务，这些服务不再公用`DB`等
